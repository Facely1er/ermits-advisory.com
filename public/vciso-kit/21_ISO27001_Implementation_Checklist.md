## ISO 27001 Overview

ISO 27001 is the international standard for Information Security Management Systems (ISMS).
The standard includes:
- 4 clauses on organizational requirements
- 14 control categories
- 93 individual controls across these categories

## Phase 2: Risk Assessment & Treatment (Months 2-3)

### 2.1 Asset Identification

- [ ] Information assets inventoried
  - [ ] Hardware/IT systems
  - [ ] Software/applications
  - [ ] Data/information
  - [ ] Personnel/expertise
  - [ ] Physical assets
  - [ ] Services/processes
- [ ] Asset owners assigned
- [ ] Asset sensitivity/criticality assessed
- [ ] Asset documentation completed

### 2.2 Threat & Vulnerability Assessment

- [ ] Threats to organization identified
- [ ] Vulnerabilities identified
  - [ ] Technical vulnerabilities
  - [ ] Process vulnerabilities
  - [ ] Personnel vulnerabilities
- [ ] Threat-asset combinations documented
- [ ] Likelihood and impact assessed for each combination

### 2.3 Risk Assessment

- [ ] Risk assessment methodology defined
- [ ] Risk calculation formula established
- [ ] Risks scored for all assets
- [ ] Risks ranked by priority
- [ ] Risk treatment options identified
  - [ ] Mitigate (reduce risk through controls)
  - [ ] Accept (accept the risk)
  - [ ] Avoid (eliminate the risk)
  - [ ] Transfer (shift risk to third party)
- [ ] Risk assessment report documented

### 2.4 Risk Treatment

- [ ] Treatment strategy for each significant risk
- [ ] Controls selected to mitigate risks
- [ ] Controls documented in Statement of Applicability (SoA)
- [ ] Controls mapped to ISO 27001 objectives
- [ ] Alternative/compensating controls identified
- [ ] Risk reduction targets set
- [ ] Treatment plan approved by management

### 2.5 Statement of Applicability (SoA)

- [ ] Reviewed all 93 ISO 27001 controls
- [ ] Determined applicability for each control
- [ ] For applicable controls: documented as mandatory
- [ ] For non-applicable controls: documented with justification
- [ ] Risk treatment mapped to controls
- [ ] SoA document completed and approved
- [ ] SoA signed by leadership

## Phase 6: Control Implementation - Assets & Access (Months 6-8)

### A.7: Asset Management Controls

**A.7.1 - Asset Inventory**
- [ ] All information assets documented
- [ ] Assets categorized
- [ ] Asset owners assigned
- [ ] Asset criticality rated
- [ ] Asset locations documented
- [ ] Asset disposal procedures defined

**A.7.2 - Information Classification**
- [ ] Classification scheme established
- [ ] Classification levels defined
- [ ] Information classified per scheme
- [ ] Classification documented
- [ ] Re-classification process
- [ ] Handling requirements for each level

**A.7.3 - Media Handling**
- [ ] Media types identified
- [ ] Use restrictions defined
- [ ] Storage requirements defined
- [ ] Transport requirements
- [ ] Disposal procedures documented
- [ ] Secure disposal implemented

### A.8: Access Control

**A.8.1 - User Access Management**
- [ ] Access control policy documented
- [ ] Access request process defined
- [ ] Access approval by manager/owner
- [ ] Access provisioning procedures
- [ ] Access review procedures (quarterly minimum)
- [ ] Access removal at termination
- [ ] Access revocation procedures

**A.8.2 - Privileged Access Rights**
- [ ] Privileged access documented
- [ ] Principle of least privilege applied
- [ ] Privileged access restricted to need-to-know
- [ ] Privileged accounts monitored
- [ ] Privileged access reviewed regularly
- [ ] Segregation of duties enforced

**A.8.3 - Restriction of Access**
- [ ] Access based on least privilege
- [ ] Access controls documented
- [ ] Access controls enforced technically
- [ ] Access decisions documented
- [ ] Access effectiveness reviewed
- [ ] Exceptions approved and tracked

**A.8.4 - Management of Secret Authentication**
- [ ] Password policy established
- [ ] Complexity requirements defined
- [ ] Password history enforced
- [ ] Password expiration set
- [ ] Password reuse prevented
- [ ] Passwords not written down
- [ ] Password resets documented

**A.8.5 - Access Rights Review**
- [ ] Access reviews performed (minimum quarterly)
- [ ] Current access vs. needed access compared
- [ ] Excessive access identified
- [ ] Unused access removed
- [ ] Review results documented
- [ ] Exceptions approved

**A.8.6 - Removal/Adjustment of Access**
- [ ] Access removal procedures documented
- [ ] Removal occurs at termination
- [ ] All access removed (systems, physical, etc.)
- [ ] Removal documented
- [ ] Equipment returned
- [ ] Off-boarding checklist used
