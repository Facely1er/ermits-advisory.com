## GDPR Overview

The GDPR establishes requirements for:
- Lawful, fair, transparent processing
- Purpose limitation
- Data minimization
- Accuracy
- Storage limitation
- Security
- Accountability

**Applies if**:
- Your organization processes personal data of EU residents, OR
- You offer goods/services to EU residents, OR
- You monitor behavior of EU residents

- [ ] Legal basis for processing identified (Article 6)

  - [ ] Contract, OR
  - [ ] Legal obligation, OR
  - [ ] Vital interests, OR
  - [ ] Public task, OR
  - [ ] Legitimate interests

- [ ] Fairness assessed
- [ ] Transparency ensured (privacy notices provided)

**Purpose Limitation**:
- [ ] Purposes for processing documented
- [ ] Personal data only used for specified purposes
- [ ] Incompatible purposes identified
- [ ] Additional purposes require new legal basis

**Data Minimization**:
- [ ] Minimum necessary data collected
- [ ] Unnecessary data not collected
- [ ] Data collection scope reviewed
- [ ] Excess data identified and minimized

**Accuracy**:
- [ ] Data accuracy ensured
- [ ] Inaccurate data identified and corrected
- [ ] Outdated data updated
- [ ] Process for keeping data accurate

**Storage Limitation**:
- [ ] Data retention periods defined
- [ ] Data retention policy documented
- [ ] Data disposed of after retention period
- [ ] Retention justified for each data type

**Integrity and Confidentiality**:
- [ ] Security measures implemented (Article 32)
- [ ] Encryption used for sensitive data
- [ ] Access controls in place
- [ ] Monitoring for breaches

**Accountability**:
- [ ] Accountability measures in place
- [ ] Documentation of processing
- [ ] Records of data processing activities
- [ ] Impact assessments performed

## PART 2: DATA SUBJECT RIGHTS

### Articles 12-22: Rights of Data Subjects

**Article 12 - Transparent Information, Communication & Modalities**
- [ ] Information provided in transparent manner
- [ ] Easy to understand format
- [ ] Plain language used
- [ ] Information provided without undue delay

**Article 13 - Information to Be Provided When Data Collected**
- [ ] Privacy notice provided at collection
- [ ] Notice includes:
  - [ ] Identity of controller
  - [ ] Purposes of processing
  - [ ] Legal basis
  - [ ] Recipients of data
  - [ ] Retention period
  - [ ] Data subject rights
  - [ ] Complaint procedures
  - [ ] Automated decision-making information (if applicable)
  - [ ] Source of data (if not from subject)

**Article 14 - Information to Be Provided When Data Not Directly Collected**
- [ ] Information provided if data from other sources
- [ ] Notice provided within reasonable period
- [ ] Includes all information from Article 13

**Article 15 - Right of Access**
- [ ] Process for subject access requests established
- [ ] Requests fulfilled within 30 days
- [ ] Copy of data provided
- [ ] No fee (unless excessive requests)
- [ ] System capable of providing data

**Article 16 - Right to Rectification**
- [ ] Process for correction requests
- [ ] Inaccurate data corrected
- [ ] Incomplete data completed
- [ ] Process documented and tracked

**Article 17 - Right to Erasure**
- [ ] Process for deletion/erasure requests
- [ ] Data deleted when no longer needed
- [ ] Exceptions to erasure understood:
  - [ ] Legal obligations
  - [ ] Public interest
  - [ ] Legitimate interests
- [ ] System capable of deleting data
- [ ] Tracking of erasure fulfillment

**Article 18 - Right to Restrict Processing**
- [ ] Process for restricting processing
- [ ] Processing restricted when requested
- [ ] Data retained but not actively processed
- [ ] Ability to restrict in systems

**Article 19 - Notification Obligation**
- [ ] Data subjects notified of corrections/erasures
- [ ] Recipients notified (if data shared)
- [ ] Exception: burdensome notification

**Article 20 - Right to Data Portability**
- [ ] Process for data portability requests
- [ ] Data provided in structured, commonly used format
- [ ] Data provided in machine-readable format
- [ ] Direct transmission to other controller (if requested)
- [ ] Systems capable of exporting data

**Article 21 - Right to Object**
- [ ] Process for objection requests
- [ ] Right to object to legitimate interest processing
- [ ] Right to object to direct marketing
- [ ] Objections honored promptly
- [ ] System to track and implement objections

**Article 22 - Automated Individual Decision-Making**
- [ ] No solely automated decisions with legal/significant effects
- [ ] If automated decisions: human review required
- [ ] Data subject right to explanation
- [ ] Data subject right to challenge decision
- [ ] Safeguards implemented

## PART 5: SECURITY & REPORTING

**Article 32 - Security of Processing**
- [ ] Security assessment performed
- [ ] Appropriate security measures:
  - [ ] Pseudonymization
  - [ ] Encryption
  - [ ] Ability to maintain confidentiality/integrity
  - [ ] Ability to restore availability
  - [ ] Process to test security measures
  - [ ] Monitor for vulnerabilities
  - [ ] Staff training
  - [ ] Staff authorized access only
  - [ ] Secure supplier relationships
  - [ ] Secure data transfer

**Article 33 - Breach Notification**
- [ ] Breach definition understood
- [ ] Breach notification process established
- [ ] Notification to supervisory authority within 72 hours
- [ ] Notification includes:
  - [ ] Nature of breach
  - [ ] Categories of data
  - [ ] Likely consequences
  - [ ] Measures taken/proposed
  - [ ] DPO/contact information
- [ ] System to detect and report breaches
- [ ] Procedures documented

**Article 34 - Communication to Data Subjects**
- [ ] Notification to data subjects of high-risk breaches
- [ ] Communication without undue delay
- [ ] Plain language
- [ ] Includes:
  - [ ] Name of DPO
  - [ ] Description of breach
  - [ ] Likely consequences
  - [ ] Measures taken/proposed
  - [ ] Remedies available
- [ ] Notification method appropriate

- [ ] Transfer impact assessments
